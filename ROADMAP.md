# MoltMine Roadmap

This roadmap is intentionally lightweight. We optimize for: **shipping playable slices**, keeping interfaces stable, and inviting contribution.

## Phase 0 — Scaffolding (now)
- [ ] Define the World API surface (actions, events, snapshots)
- [ ] Define entity/component schema + versioning rules
- [ ] Establish contribution standards + security policy

## Phase 1 — Joinable world (MVP)
- [ ] Auth + presence
- [ ] Authoritative movement
- [ ] Persistence (basic)
- [ ] Minimal web client

## Phase 2 — Building + claims
- [ ] Object placement + editing
- [ ] Claims/permissions/ACLs
- [ ] Audit logs + moderation hooks

## Phase 3 — Moltys agents v1
- [ ] Action API adapter for agents
- [ ] Safe tool/capability sandbox
- [ ] Evaluation harness

## Phase 4 — Sim-to-live loop
- [ ] Simulation environment mirroring core interactions
- [ ] Reproducible scenarios + metrics
- [ ] Canary deployments
